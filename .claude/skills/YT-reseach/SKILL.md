# Diagram Garden

ガーデニング×YouTubeの企画立案・台本作成に必要な情報を素早く視覚化する個人用図解スキル。

**ゴール**: 図解を見た後、「次の動画テーマ」と「その根拠」が即座に言える状態にする。

## チャンネル情報・ベンチマーク

作業開始前に必ず読む → [references/channel-info.md](references/channel-info.md)

- 自チャンネルのコンセプト・ターゲット・取り扱わないトピックが記載
- ベンチマークYouTuberリストが記載（ベンチマーク分析時はここから対象を選ぶ）

## ターゲット情報

企画アイデア作成・図解の言葉選び時に参照 → [references/target-info.md](references/target-info.md)

- グリーンロケットのターゲット3層（手入れ改善層・近隣配慮層・DIY初心者層）
- 層別の悩み・検索キーワード・検索意図
- 具体的ペルソナ3名（香織38・修45・麻衣32）
- タイトルに刺さるキーワードと企画切り口のメモ

### Phase 0.5: YouTube API ショートデータ収集（shell サブエージェント）

Phase 0（explore サブエージェント）の直後、Phase 1 の前に実行する。
`scripts/youtube_api_research.py` を使い、ショートの実データ（再生数・尺）を API で取得する。
取得結果 JSON は Phase 1 のまとめに統合すること。

**前提条件**: `YT-reseach/.env` に `YOUTUBE_API_KEY=<実際のキー>` が設定されていること。
設定されていない場合はこのフェーズをスキップし、Phase 0 の Web 検索データのみで進める。

```bash
# 依存パッケージをインストール（初回のみ）
pip install -r scripts/requirements.txt -q

# 直近1ヶ月のショート（今月）
python scripts/youtube_api_research.py \
  --month {公開月} \
  --year {今年} \
  --category recent_shorts \
  --output output/api_shorts_recent_{月}.json

# 過去5年・{公開月}月公開のショート
python scripts/youtube_api_research.py \
  --month {公開月} \
  --years 2021 2022 2023 2024 2025 2026 \
  --category past_shorts \
  --output output/api_shorts_past_{月}.json

# 小規模チャンネルのショートバズ（7万人以下・過去7年同月）
python scripts/youtube_api_research.py \
  --month {公開月} \
  --years 2021 2022 2023 2024 2025 2026 \
  --category small_channel_shorts \
  --max-subs 70000 \
  --output output/api_shorts_small_historical_{月}.json

# 小規模チャンネルのショートバズ（7万人以下・直近2ヶ月）
python scripts/youtube_api_research.py \
  --month {今月} \
  --year {今年} \
  --category small_channel_shorts \
  --max-subs 70000 \
  --output output/api_shorts_small_recent_{今月}.json
python scripts/youtube_api_research.py \
  --month {先月} \
  --year {先月の年} \
  --category small_channel_shorts \
  --max-subs 70000 \
  --output output/api_shorts_small_recent_{先月}.json

# ジャンル外チャンネルのショートバズ
python scripts/youtube_api_research.py \
  --month {公開月} \
  --years 2021 2022 2023 2024 2025 2026 \
  --category genre_out_shorts \
  --output output/api_shorts_genreout_{月}.json
```

**出力 JSON の使い方**: `output/api_shorts_*.json` の `shorts` 配列を Phase 1 の各ショートリストに
マージし、`viewCount` でソートして上位10本に絞る。
Web 検索で見つけた動画と重複する場合は API 取得の `viewCount` 実数値を優先する。

### Phase 1: 情報を整理する

Phase 0の結果から以下を抽出する：

1. **即使える発見**（企画に直結する情報）
2. **トレンド方向**（↑上昇 / → 安定 / ↓下降）
3. **直近1ヶ月の伸び動画リスト**（常時）以下の形式で整理する：
   - 動画タイトル / チャンネル名（YouTuber名）/ 尺（ショート or ロング）/ 推定再生数
   - **ショート（60秒以下）上位10本・ロング（3分超）上位10本を別リストで出力**
   - 見つからなければ判明した分のみ記載
4. **小規模チャンネル バズ動画リスト A（過去7年・同月）**（常時）以下の形式で整理する：
   - 動画タイトル / チャンネル名 / 登録者数（7万人以下）/ 投稿年月 / 尺（ショート or ロング）/ 再生数（推定）/ なぜ伸びているかの仮説
   - 対象：過去7年間の今と同じ月（今年含む）に投稿された動画
   - **ショート最大10本・ロング最大10本を別リストで出力（ショートは2021年以降のみ）**
4b. **小規模チャンネル バズ動画リスト B（直近2ヶ月・急伸中）**（常時）以下の形式で整理する：
   - 動画タイトル / チャンネル名 / 登録者数（7万人以下）/ 投稿年月 / 尺（ショート or ロング）/ 再生数（推定）/ なぜ今伸びているかの仮説
   - 対象：直近2ヶ月以内に投稿され、現在進行形でバズっている動画
   - **ショート最大10本・ロング最大10本を別リストで出力**
5. **ジャンル外チャンネルのバズ動画リスト**（常時）農業・ガーデニング・園芸系チャンネルが植木・庭木テーマで投稿して異常に伸びている動画。以下の形式で整理する：
   - 動画タイトル / チャンネル名（メインジャンル）/ 尺（ショート or ロング）/ 再生数（推定）/ なぜ伸びているかの仮説
   - **ショート最大10本・ロング最大10本を別リストで出力**
6. **過去の同時期人気動画リスト**（公開時期指定時）以下の形式で整理する：
   - 動画タイトル / チャンネル名 / 投稿年月 / 尺（ショート or ロング）/ 再生数（推定）/ なぜ伸びたか仮説
   - **ロング最大10本・ショート最大10本を別リストで出力（ショートは2021年以降のみ）**
   - タイトル共通パターンと「毎年伸びるテーマ」vs「その年だけのテーマ」も記載
7. **売れ筋植物リスト**（時期指定がある場合）カテゴリ別に列挙する
   - 苗 / 植木 / 低木 / 宿根草
   - 各植物に「売れる理由・需要の背景」を添える
7. **おすすめ企画アイデア**（以下の3カテゴリに分けて各2〜3案、各案に根拠を添える）

#### 企画アイデアの3カテゴリ

企画アイデアは必ず以下の3つに分類して提案する：

**① 作業系**
その時期に発生する庭仕事・植木管理の作業テーマ。
- 剪定・切り戻し・花がら摘み
- 除草・防草
- 害虫・病気対策
- 水やり・施肥・マルチング
- 植え替え・移植

**② お庭におすすめの植物**
その時期に植え付け・購入・開花・管理の観点から初心者に紹介しやすい植物。
植物カテゴリ別に考える（該当するものを選ぶ）：
- 高木（シンボルツリー等）
- 低木（アジサイ・ツツジ等）
- 下草・一年草（ペチュニア・マリーゴールド等）
- 宿根草（繰り返し咲く・管理が楽な植物）
- ハーブ

**③ イベント・特集系**
特定テーマの特集、品種紹介、シリーズもの、お役立ち情報など。
- 品種・種類紹介（例：ツツジ品種紹介、アジサイ品種5選）
- 特集動画（例：梅雨前庭仕事チェックリスト、ドクダミ徹底対策）
- ランキング・まとめ（例：初心者に育てやすい木5選）
- プロ視点の知識共有（例：造園業者が教える意外な常識）

**取り扱わないトピックの除外確認**：
- 「バラ」「一年草花苗（ペチュニア・マリーゴールド・パンジー・ビオラ等）」「野菜苗（トマト・ナス・ピーマン・キュウリ等）」が企画案に含まれていたら除き、代替テーマを提案する
- 詳細 → [references/channel-info.md](references/channel-info.md)

図解タイプを判定して構成を決める。

**詳細** → [references/diagram-types.md](references/diagram-types.md)

### Phase 2: HTML生成

**詳細** → [references/html-structure.md](references/html-structure.md)

基本ルール：
- Tailwind CSS CDN 使用
- Lucide icon 使用（絵文字禁止）
- ガーデンカラーパレット適用（green-800/green-600 系統）
- 必ず以下の4つのリサーチテーブルセクションを含める（企画アイデアセクションの直前にこの順番で配置）

  **① 直近1ヶ月 伸びている動画**
  - **ショート（60秒以下）とロング（3分超）を2段に分けて表示**
  - [ショート] ヘッダー行を pink-600 背景・白文字
  - [ロング]   ヘッダー行を green-800 背景・白文字
  - 列構成: # / 動画タイトル / チャンネル名 / 尺（推定）/ 再生数（推定）
  - 行数: 各最大10行（見つかった分のみ）
  - 奇数行を各テーマ色の-50 背景

  **② 小規模チャンネル バズ動画 A**（登録者7万人以下・過去7年の同月・チャンネル平均を大きく上回る動画）
  - **ショートとロングを2段に分けて表示**
  - [ショート] ヘッダー行を pink-600 背景・白文字（2021年以降のみ）
  - [ロング]   ヘッダー行を amber-700 背景・白文字
  - 列構成: # / 動画タイトル / チャンネル名 / 登録者数（7万人以下）/ 投稿年月 / 尺（推定）/ 再生数（推定）/ なぜ伸びたか
  - 行数: 各最大10行（見つかった分のみ）

  **②-b 小規模チャンネル バズ動画 B**（登録者7万人以下・直近2ヶ月投稿・今まさに急伸中の動画）
  - **ショートとロングを2段に分けて表示**
  - [ショート] ヘッダー行を pink-600 背景・白文字
  - [ロング]   ヘッダー行を orange-600 背景・白文字
  - 列構成: # / 動画タイトル / チャンネル名 / 登録者数（7万人以下）/ 投稿年月 / 尺（推定）/ 再生数（推定）/ なぜ今伸びているか
  - 行数: 各最大10行（見つかった分のみ）

  **③ ジャンル外 バズ動画**（農業・ガーデニング・園芸系チャンネルが植木テーマで伸びている動画）
  - **ショートとロングを2段に分けて表示**
  - [ショート] ヘッダー行を pink-600 背景・白文字
  - [ロング]   ヘッダー行を blue-700 背景・白文字
  - 列構成: # / 動画タイトル / チャンネル名（メインジャンル）/ 尺（推定）/ 再生数（推定）/ なぜ伸びたか
  - 行数: 各最大10行（見つかった分のみ）

  **④ 過去5年・同時期公開の人気動画**（公開時期指定時のみ）
  - **ショートとロングを2段に分けて表示**
  - [ロング]   ヘッダー行を gray-600 背景・白文字（2020年以前を含む全期間）
  - [ショート] ヘッダー行を pink-600 背景・白文字（2021年以降のみ）
  - 列構成: # / 動画タイトル / チャンネル名 / 投稿年月 / 尺（推定）/ 再生数（推定）/ なぜ伸びたか
  - 行数: 各最大10行（見つかった分のみ）
  - テーブルの後に「毎年繰り返し伸びているテーマ」パターンカードとタイトル共通パターンバッジを配置

- 必ず「企画アイデアセクション」で締める
- インタラクティブ要素禁止（アニメーション・フォームなし）
- `<style>` タグ追加禁止（スタイリングは Tailwind クラスで）

ファイルを `output/{スラッグ}.html` に保存する。

### Phase 3: 批判的レビュー（サブエージェント）

**YouTubeコンテンツ制作者の視点**で、忖度なくレビューする。

```
Task({
  subagent_type: "generalPurpose",
  description: "図解の批判的レビュー",
  readonly: true,
  prompt: `
    あなたはガーデニング系YouTubeチャンネルを個人運営しているコンテンツクリエイターです。
    月に2〜3本動画を出しており、毎回テーマ選びと台本作成に時間をかけています。
    チャンネル登録者は数千〜数万人規模。

    以下のHTML図解を読んで、忖度なく批判してください：

    【HTMLファイルパス】{path}

    ## レビュー観点

    1. **企画に使えるか**: この図解を見て、次の動画テーマが決められるか？
    2. **根拠はあるか**: 「なぜそのテーマが今良いか」の根拠が示されているか？
    3. **スキャンしやすいか**: 30秒でざっと見て要点が掴めるか？
    4. **情報の抜け漏れ**: 企画立案に必要なのに欠けている情報は？
    5. **企画アイデアの具体性**: 提案された企画は台本を書き始められるレベルか？

    ## 出力形式

    ### 致命的な問題（このままでは企画に使えない）
    - ...

    ### 改善すべき点
    - ...

    ### 良かった点
    - ...

    ### 総評
    5段階評価: ⭐⭐⭐☆☆（3/5）のように
    一言: 「○○だから△△な企画に使える/使えない」のように
  `
})
```

### Phase 4: ブラッシュアップ

Phase 3のレビュー結果を踏まえてHTMLを修正する。

**優先度**：
1. 致命的な問題 → 必ず修正
2. 改善すべき点 → 可能な限り対応
3. 良かった点 → 維持

### Phase 5: デプロイ

surge.sh は**ディレクトリ単位**でデプロイするため、HTMLを `output/deploy/index.html` にコピーしてから実行する。

**Windows（PowerShell）の場合：**
```powershell
# 1. deploy ディレクトリに index.html としてコピー
Copy-Item "output/{スラッグ}.html" "output/deploy/index.html" -Force

# 2. surge でディレクトリをデプロイ（作業ディレクトリは output/ に移動してから実行）
cd output
npx surge deploy --domain {スラッグ}.surge.sh
```

**macOS / Git Bash の場合：**
```bash
# 1. deploy ディレクトリに index.html としてコピー
cp output/{スラッグ}.html output/deploy/index.html

# 2. surge でディレクトリをデプロイ
cd output && npx surge deploy --domain {スラッグ}.surge.sh
```

- 初回はメールアドレスとパスワードの入力が必要。次回以降は不要。
- `{スラッグ}` は HTMLファイル名のベース部分（例: `planning-jun2026-v3`）を使う。
- ドメインはプロジェクトルーティングに合わせて決定（例: `green-rocket-jun2026.surge.sh`）。

デプロイ完了後、以下の形式で報告する：

```
完成・公開完了: 【図解のタイトル】

【図解の内容を1〜2文で要約】

公開URL:
https://{スラッグ}.surge.sh

企画アイデア（上位3案）:
1. 【タイトル案】— 根拠：【なぜ今か】
2. 【タイトル案】— 根拠：【なぜ今か】
3. 【タイトル案】— 根拠：【なぜ今か】
```
