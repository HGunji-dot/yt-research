# Diagram Garden

ガーデニング×YouTubeの企画立案・台本作成に必要な情報を素早く視覚化する個人用図解スキル。

**ゴール**: 図解を見た後、「次の動画テーマ」と「その根拠」が即座に言える状態にする。

## チャンネル情報・ベンチマーク

作業開始前に必ず読む → [references/channel-info.md](references/channel-info.md)

- 自チャンネルのコンセプト・ターゲット・取り扱わないトピックが記載
- ベンチマークYouTuberリストが記載（ベンチマーク分析時はここから対象を選ぶ）

## ターゲット情報

企画アイデア作成・図解の言葉選び時に参照 → [references/target-info.md](references/target-info.md)

- グリーンロケットのターゲット3層（手入れ改善層・近隣配慮層・DIY初心者層）
- 層別の悩み・検索キーワード・検索意図
- 具体的ペルソナ3名（香織38・修45・麻衣32）
- タイトルに刺さるキーワードと企画切り口のメモ

## ワークフロー

### Phase 0: リサーチ（サブエージェント）

exploreサブエージェントを起動して対象トピックを調査する。

```
Task({
  subagent_type: "explore",
  description: "ガーデニング/YouTubeトピック調査",
  prompt: `
    以下のトピックについて調査してください：
    【対象】{トピック}

    調査項目（依頼内容に応じて選択）：

    ## トレンド分析の場合
    1. 国内のガーデニングトレンド（直近1〜3ヶ月）
    2. 季節要因（今後1〜2ヶ月の見込み）
    3. 上昇中・安定・下降トレンドの分類と根拠
    ※ 調査結果から「バラ」「一年草花苗（ペチュニア・マリーゴールド・パンジー等）」「野菜苗（トマト・ナス・ピーマン・キュウリ等）」に関する情報は除外してください

    ## ベンチマーク分析の場合
    以下のチャンネルの動向を調査してください（ベンチマークリスト）：
    - Carmen-kun Garden Channel: https://www.youtube.com/@Kahmen-Gardening
    - Garden Designer ch グリーンギャラリーガーデンズ: https://www.youtube.com/@gardendesignerch2502
    - かべちょろGardenClass: https://www.youtube.com/@%E3%81%8B%E3%81%B9%E3%81%A1%E3%82%87%E3%82%8D
    - yamakana farm やまかなふぁーむ: https://www.youtube.com/@yamakanafarm8217
    - Garden Produce TV / いろは造園: https://www.youtube.com/@irohazoen
    - ガーデンセンターさにべる: https://www.youtube.com/@sunnyvale-gc.8783

    調査内容：
    1. 対象チャンネルの直近5〜10本の動画タイトルと再生数（推定でも可）
    2. 伸びている動画のテーマパターン
    3. 投稿頻度・最近の方向性の変化

    ## キーワード分析の場合
    1. 対象キーワードの季節性（例年の動向）
    2. 関連キーワード・共起キーワード一覧
    3. 競合コンテンツの傾向（検索結果の概観）

    ## 売れ筋植物リスト調査（公開時期指定時は必ず実施）
    指定した時期にホームセンター・園芸店・通販で売れている植物を調査する。

    調査カテゴリ：
    - 苗（草花苗・宿根草苗）
    - 植木（シンボルツリー・庭木）
    - 低木（アジサイ・ツツジ・ブルーベリー等）
    - 宿根草（クリスマスローズ・エキナセア・サルビア等）

    調査手順:
    1. Google 検索: 「{月} 植木 人気 売れ筋 園芸」「{月} 庭木 おすすめ ホームセンター」
    2. 大手通販（Amazon・楽天・Yahoo!ショッピング）の園芸カテゴリで時期別ランキングを確認
    3. SNS（Instagram・X）で「#ガーデニング #{月}」「#{植物名} 植え付け」の投稿数・反応を確認

    まとめ方:
    - カテゴリ別に植物名と売れる理由（季節性・管理のしやすさ・見栄えなど）を列挙
    - 「毎年この時期に必ず需要がある定番」と「今年特に注目されている品種」を分類
    - 取り扱わないトピック（バラ・野菜苗・一年草花苗）は除外する

    ## 過去の人気動画調査の場合（公開時期指定時は必ず実施）
    指定した公開時期（例: 5月2週目）に過去5年間で再生数・反応が多かった
    ガーデニング系YouTube動画のパターンを調査する。

    調査手順:
    1. Google 検索: 「ガーデニング YouTube 5月 人気動画 site:youtube.com」
       「植木 庭 YouTube 5月 再生数 ランキング」など複数クエリで検索
    2. 各ベンチマークチャンネルの過去5月投稿動画を確認
       （チャンネルURLに /videos?view=0&sort=p でアクセスすると再生数順になる）
    3. Google Trends でガーデニング関連ワードの5月の例年ピークを確認

    まとめ方:
    - 動画タイトル（または推定テーマ）、チャンネル名、推定再生数・年を列挙
    - タイトルパターンの共通点を抽出（「失敗しない」「今すぐ」「〜選」など）
    - 「同じ時期に毎年伸びているテーマ」と「その年だけのテーマ」を分類

    ## 直近1ヶ月の伸び動画調査（常時実施・依頼内容に関わらず必ず行う）
    ガーデニング系YouTubeチャンネル全体で直近1ヶ月に再生数が伸びた動画を調査する。
    ベンチマーク6社に限らず、ガーデニング・植木・造園ジャンル全般を対象とする。

    調査手順:
    1. YouTube検索: 「ガーデニング」「植木」「庭づくり」で投稿日フィルター（1ヶ月以内）をかけ、再生数上位の動画を確認
    2. Google検索: 「ガーデニング YouTube 人気動画 {今月}」「植木 庭 YouTube {今月} 再生数」
    3. 各ベンチマークチャンネルの直近1ヶ月の動画で再生数が多いものを確認

    まとめ方:
    - 動画タイトル・チャンネル名（YouTuber名）・推定再生数を列挙
    - 件数: 上位5〜10本
    - 再生数は実数が取得できない場合は「数万回」「数千回」など概算で記載し、取得不可の場合は「取得不可」と記載

    ## 小規模チャンネルのバズ動画調査（常時実施・依頼内容に関わらず必ず行う）
    以下の3条件をすべて満たす植木・ガーデニング系YouTube動画を調査する：
    1. 登録者数が7万人以下のチャンネルが投稿した動画
    2. そのチャンネルの平均再生数と比べて明らかに突出している動画
    3. 投稿時期が過去7年間の今と同じ月（今年含む）の動画
       例：調査時が2026年5月なら、2020〜2026年の各5月に投稿された動画が対象

    調査手順:
    1. YouTube検索: 「植木」「庭木」「ガーデニング」で検索し、登録者数7万人以下のチャンネルで
       再生数が突出している動画を探す（投稿日フィルターで絞り込みながら過去7年分を確認）
    2. Google検索: 「ガーデニング YouTube {同月} 再生数 人気」「植木 YouTube {同月} バズ 小規模」
    3. 各ベンチマークチャンネルで登録者7万人以下のものについて、過去7年間の今と同じ月の動画の中で
       チャンネル平均再生数を大きく上回っているものを確認

    まとめ方:
    - 動画タイトル・チャンネル名・登録者数（推定）・投稿年月・再生数（推定）・「なぜ伸びているか」の仮説を列挙
    - 件数: 3〜5本
    - 仮説は「タイトルの引きが強い」「季節ニーズと合致」「検索上位を取れるキーワード」など1行で

    ## ジャンル外チャンネルのバズ動画調査（常時実施・依頼内容に関わらず必ず行う）
    造園・植木専門チャンネルではなく、農業・ガーデニング・園芸系チャンネルが
    植木・庭木テーマで投稿して異常に伸びている動画を調査する。

    調査手順:
    1. YouTube検索: 「植木」「庭木」「シンボルツリー」などで検索し、農業・ガーデニング・園芸系チャンネルの動画を探す
    2. Google検索: 「ガーデニング YouTube 植木 人気」「農業 YouTuber 庭木 動画 再生数」

    まとめ方:
    - 動画タイトル・チャンネル名（メインジャンル）・再生数（推定）・「なぜ伸びているか」の仮説を列挙
    - 件数: 3〜5本
    - 仮説は「既存ファン層に刺さった」「他ジャンルの視聴者を取り込んだ」など1行で

    ※ YouTube の正確な再生数はAPIなしでは取得不可のため推定・外部情報で代替可。
    ※ 情報が取得できなかった場合は「取得不可」と記載し推測根拠を添えること。

    結果は箇条書きで整理して返してください。
  `
})
```

### Phase 1: 情報を整理する

Phase 0の結果から以下を抽出する：

1. **即使える発見**（企画に直結する情報）
2. **トレンド方向**（↑上昇 / → 安定 / ↓下降）
3. **直近1ヶ月の伸び動画リスト**（常時）以下の形式で整理する：
   - 動画タイトル / チャンネル名（YouTuber名）/ 推定再生数
   - 上位5〜10本を列挙
4. **小規模チャンネルのバズ動画リスト**（常時）以下の形式で整理する：
   - 動画タイトル / チャンネル名 / 登録者数（7万人以下）/ 投稿年月 / 再生数（推定）/ なぜ伸びているかの仮説
   - 対象：過去7年間の今と同じ月（今年含む）に投稿された動画
   - 3〜5本を列挙
5. **ジャンル外チャンネルのバズ動画リスト**（常時）農業・ガーデニング・園芸系チャンネルが植木・庭木テーマで投稿して異常に伸びている動画。以下の形式で整理する：
   - 動画タイトル / チャンネル名（メインジャンル）/ 再生数（推定）/ なぜ伸びているかの仮説
   - 3〜5本を列挙
6. **売れ筋植物リスト**（時期指定がある場合）カテゴリ別に列挙する
   - 苗 / 植木 / 低木 / 宿根草
   - 各植物に「売れる理由・需要の背景」を添える
7. **おすすめ企画アイデア**（以下の3カテゴリに分けて各2〜3案、各案に根拠を添える）

#### 企画アイデアの3カテゴリ

企画アイデアは必ず以下の3つに分類して提案する：

**① 作業系**
その時期に発生する庭仕事・植木管理の作業テーマ。
- 剪定・切り戻し・花がら摘み
- 除草・防草
- 害虫・病気対策
- 水やり・施肥・マルチング
- 植え替え・移植

**② お庭におすすめの植物**
その時期に植え付け・購入・開花・管理の観点から初心者に紹介しやすい植物。
植物カテゴリ別に考える（該当するものを選ぶ）：
- 高木（シンボルツリー等）
- 低木（アジサイ・ツツジ等）
- 下草・一年草（ペチュニア・マリーゴールド等）
- 宿根草（繰り返し咲く・管理が楽な植物）
- ハーブ

**③ イベント・特集系**
特定テーマの特集、品種紹介、シリーズもの、お役立ち情報など。
- 品種・種類紹介（例：ツツジ品種紹介、アジサイ品種5選）
- 特集動画（例：梅雨前庭仕事チェックリスト、ドクダミ徹底対策）
- ランキング・まとめ（例：初心者に育てやすい木5選）
- プロ視点の知識共有（例：造園業者が教える意外な常識）

**取り扱わないトピックの除外確認**：
- 「バラ」「一年草花苗（ペチュニア・マリーゴールド・パンジー・ビオラ等）」「野菜苗（トマト・ナス・ピーマン・キュウリ等）」が企画案に含まれていたら除き、代替テーマを提案する
- 詳細 → [references/channel-info.md](references/channel-info.md)

図解タイプを判定して構成を決める。

**詳細** → [references/diagram-types.md](references/diagram-types.md)

### Phase 2: HTML生成

**詳細** → [references/html-structure.md](references/html-structure.md)

基本ルール：
- Tailwind CSS CDN 使用
- Lucide icon 使用（絵文字禁止）
- ガーデンカラーパレット適用（green-800/green-600 系統）
- 必ず以下の3つのリサーチテーブルセクションを含める（企画アイデアセクションの直前にこの順番で配置）

  **① 直近1ヶ月 伸びている動画**
  - 列構成: # / 動画タイトル / チャンネル名 / 再生数（推定）
  - 行数: 5〜10本
  - スタイル: ヘッダー行を green-800 背景・白文字、奇数行を green-50 背景

  **② 小規模チャンネル バズ動画**（登録者7万人以下・過去7年の同月・チャンネル平均を大きく上回る動画）
  - 列構成: # / 動画タイトル / チャンネル名 / 登録者数（7万人以下）/ 投稿年月 / 再生数（推定）/ 仮説
  - 行数: 3〜5本
  - スタイル: ヘッダー行を amber-700 背景・白文字、奇数行を amber-50 背景

  **③ ジャンル外 バズ動画**（農業・ガーデニング・園芸系チャンネルが植木テーマで伸びている動画）
  - 列構成: # / 動画タイトル / チャンネル名（メインジャンル）/ 再生数（推定）/ 仮説
  - 行数: 3〜5本
  - スタイル: ヘッダー行を blue-700 背景・白文字、奇数行を blue-50 背景

- 必ず「企画アイデアセクション」で締める
- インタラクティブ要素禁止（アニメーション・フォームなし）
- `<style>` タグ追加禁止（スタイリングは Tailwind クラスで）

ファイルを `output/{スラッグ}.html` に保存する。

### Phase 3: 批判的レビュー（サブエージェント）

**YouTubeコンテンツ制作者の視点**で、忖度なくレビューする。

```
Task({
  subagent_type: "generalPurpose",
  description: "図解の批判的レビュー",
  readonly: true,
  prompt: `
    あなたはガーデニング系YouTubeチャンネルを個人運営しているコンテンツクリエイターです。
    月に2〜3本動画を出しており、毎回テーマ選びと台本作成に時間をかけています。
    チャンネル登録者は数千〜数万人規模。

    以下のHTML図解を読んで、忖度なく批判してください：

    【HTMLファイルパス】{path}

    ## レビュー観点

    1. **企画に使えるか**: この図解を見て、次の動画テーマが決められるか？
    2. **根拠はあるか**: 「なぜそのテーマが今良いか」の根拠が示されているか？
    3. **スキャンしやすいか**: 30秒でざっと見て要点が掴めるか？
    4. **情報の抜け漏れ**: 企画立案に必要なのに欠けている情報は？
    5. **企画アイデアの具体性**: 提案された企画は台本を書き始められるレベルか？

    ## 出力形式

    ### 致命的な問題（このままでは企画に使えない）
    - ...

    ### 改善すべき点
    - ...

    ### 良かった点
    - ...

    ### 総評
    5段階評価: ⭐⭐⭐☆☆（3/5）のように
    一言: 「○○だから△△な企画に使える/使えない」のように
  `
})
```

### Phase 4: ブラッシュアップ

Phase 3のレビュー結果を踏まえてHTMLを修正する。

**優先度**：
1. 致命的な問題 → 必ず修正
2. 改善すべき点 → 可能な限り対応
3. 良かった点 → 維持

### Phase 5: デプロイ

ファイル保存後、Node.jsの有無を確認する。

```bash
node --version
```

**Windows（PowerShell）の場合：**
```powershell
npx --yes surge output/{スラッグ}.html --domain diagram-{スラッグ}.surge.sh
```

**macOS / Git Bash の場合：**
```bash
npx surge output/{スラッグ}.html --domain diagram-{スラッグ}.surge.sh
```

初回はメールアドレスとパスワードの入力が必要。次回以降は不要。

デプロイ完了後、以下の形式で報告する：

## 品質チェックリスト

### 必須

- [ ] 企画アイデアセクションが最後にある（3案以上）
- [ ] 各企画アイデアに「なぜ今か」の根拠がある
- [ ] トレンド方向が視覚的に表現されている（Lucide icon + 色分け）
- [ ] 図解タイプに応じた必須セクションが含まれている（→ diagram-types.md）
- [ ] 取り扱わないトピック（バラ・一年草花苗等）が企画案に含まれていない（→ channel-info.md）
- [ ] ターゲット（30〜50代・初心者）に刺さる内容になっているか
- [ ] 「直近1ヶ月 伸びている動画」テーブルセクションがある（5〜10本・# / タイトル / チャンネル名 / 再生数の4列）
- [ ] 「小規模チャンネル バズ動画」テーブルセクションがある（3〜5本・# / タイトル / チャンネル名 / 登録者数（7万人以下）/ 投稿年月 / 再生数 / 仮説の7列）
- [ ] 「ジャンル外 バズ動画」テーブルセクションがある（3〜5本・# / タイトル / チャンネル名（メインジャンル）/ 再生数 / 仮説の5列）

### デザイン

- [ ] Lucide iconのみ使用（絵文字なし）
- [ ] ガーデンカラーパレット適用（green-800/green-600 系統）
- [ ] スキャンしやすいレイアウト（見出し→データ→解釈の順）
- [ ] スマホでも読める（レスポンシブ）
